# Project Workflow

## Folder Convention

| Location | Purpose | Git |
|----------|---------|-----|
| `games/`, `scripts/`, `extension*/` | All code | ✓ Committed |
| `runs/kingdomino/<run_name>/` | Checkpoints, buffers, training logs | ✗ Gitignored |
| `runs/kingdomino/benchmarks/` | Throughput benchmark CSVs | ✗ Gitignored |
| `elo_db.json`, `elo_games.jsonl` | Elo ladder (repo root) | ✓ Committed |
| `games/kingdomino/best_checkpoint/` | Curated best model | ✓ Committed |

Key rule: anything generated by running code goes in `runs/`. Never
commit checkpoints, buffers, or benchmark output.

---

## Training Command Template

```powershell
python -m games.kingdomino.self_play --engine batched_open_loop --device cuda --warm_start games\kingdomino\best_checkpoint\best_32x4.pt --warm_buffer runs\kingdomino\<prev_run>\buffer_final.pkl --iterations 55 --games_per_iter 150 --train_steps 600 --sims 1600 --channels 32 --blocks 4 --batch_slots 32 --leaf_batch 6 --lr 3e-4 --buffer 300000 --lambda_score 0.5 --lambda_w 0.25 --score_scale 160 --margin_gain 2.0 --alpha 0.8 --fpu -0.2 --c_puct 1.5 --benchmark_every 10 --benchmark_sims 50 --benchmark_seeds 20 --checkpoint_dir runs\kingdomino\<run_name> --save_buffer runs\kingdomino\<run_name>\buffer_final.pkl --elo_every 10 --elo_sims 400 --elo_games_per_anchor 40 --elo_db elo_db.json --elo_games_log elo_games.jsonl --seed 0
```

Replace `<run_name>` with a descriptive name (e.g. `local_cont4`,
`cloud_48x6_run1`). Replace `<prev_run>` with the previous run's
name for the warm buffer.

## After Every Training Run

```powershell
# 1. Re-solve global Elo ladder
python -m games.kingdomino.elo_rating --resolve --verbose

# 2. If new best checkpoint, copy it
Copy-Item runs\kingdomino\<run_name>\iter_XXXX.pt games\kingdomino\best_checkpoint\best_32x4.pt

# 3. Commit Elo data and best checkpoint
git add elo_db.json elo_games.jsonl games\kingdomino\best_checkpoint\
git commit -m "Run <run_name>: <brief summary e.g. policy_loss 1.869, Elo 952>"
git push origin main
```

## Before a Cloud Run

```powershell
# 1. Ensure latest code is pushed
git push origin main

# 2. Transfer checkpoint and buffer to cloud instance
# scp runs\kingdomino\<run_name>\iter_XXXX.pt user@cloud:~/boardgame-ai/runs/kingdomino/<run_name>/
# scp runs\kingdomino\<run_name>\buffer_final.pkl user@cloud:~/boardgame-ai/runs/kingdomino/<run_name>/

# 3. On cloud: clone repo, build Rust, run throughput benchmarks first
```

---

## After a Cloud Run

```powershell
# 1. Download Elo game log and merge with local
# scp user@cloud:~/boardgame-ai/elo_games.jsonl elo_games_cloud.jsonl
# (append and deduplicate, then re-run --resolve)

# 2. Download best checkpoint if improved
# scp user@cloud:~/boardgame-ai/runs/kingdomino/<run_name>/iter_XXXX.pt games\kingdomino\best_checkpoint\best_32x4.pt

# 3. Commit
git add elo_db.json elo_games.jsonl games\kingdomino\best_checkpoint\
git commit -m "Cloud run <run_name>: <summary>"
git push origin main
```

## GitHub Push Triggers

Push after:
- Any code change (bug fix, new feature, architecture change)
- Each training run completes (Elo + best checkpoint update)
- Before any cloud run

Do not push:
- Mid-run (nothing meaningful changes)
- Checkpoint files or buffers (covered by .gitignore)
